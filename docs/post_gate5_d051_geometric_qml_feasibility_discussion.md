# D051-C Future Protocol Discussion: Geometric QML Feasibility

Version: 0.1.0  
Decision: D051-C  
Prospective protocol: P018  
Prepared: 2026-07-15  
Status: **Discussion-only; no execution authority; Gate 6 unauthorized**

## 1. Purpose and claim boundary

This document records a future test of task-specific symmetry in a variational
quantum model. It does not reopen Gate 5, refit D006-D049 evidence, authorize
calibration or final-test access, authorize hardware/GPU execution, or authorize
Gate 6.

The strongest current claim is:

> Geometric QML is a testable hypothesis for exploiting task-specific symmetry,
> not evidence of quantum advantage.

Equivariance alone is not a quantum advantage. A positive result would require
the candidate to beat both C06 and a parameter- and compute-matched classical
equivariant control under the frozen claim boundary. A simulator result would
remain a classical simulation result.

## 2. Scientific basis and limits

Group-invariant QML and symmetry-aware variational circuits provide methods for
building models that respect a known data symmetry; they do not prove a
performance or complexity advantage on this pipeline. See Larocca et al.,
"Group-Invariant Quantum Machine Learning", https://arxiv.org/abs/2205.02261,
and Meyer et al., "Exploiting symmetry in variational quantum machine
learning", https://arxiv.org/abs/2205.06217.

Classical equivariant graph networks are a required comparison, not a weak
strawman. E(n)-equivariant graph networks are an established classical model
family for geometric data; see Satorras et al.,
https://proceedings.mlr.press/v139/satorras21a.html.

Barren-plateau behavior depends on ansatz, initialization, cost function,
depth, and noise. Structured initialization and an optimizer change are
therefore empirical interventions, not guarantees. See McClean et al.,
https://www.nature.com/articles/s41467-018-07090-4, and Wang et al.,
https://www.nature.com/articles/s41467-021-27045-6.

The NPID proposal is future evidence only. Its available preprint evidence is
not a reproduction on the cislunar surrogate task and cannot be used as a
performance claim here: https://arxiv.org/abs/2511.14820.

## 3. Mathematical contract

### 3.1 Problem and data

Let `X` be the existing structured cislunar input space, decomposed only after
the symmetry audit into scalar features, vector features, body/trajectory
entities, and fixed context. Let `Y` contain the existing robust-cost target
and feasibility target. The development distribution is `P(X,Y)` generated by
the existing pipeline; no calibration or final-test row is included.

The exact group `G` is **unknown until Stage 1**. It must be derived from the
data-generating equations, ephemerides, reference frames, thrust constraints,
and output definitions. Full `SO(3)` is not assumed. A candidate transformation
`T_g: X -> X` is admissible only if the equations and all relevant constraints
are transformed consistently.

For a scalar target, the required label action is `rho_Y(g)=1` and the exact
condition is:

`y(T_g x) = y(x)`.

For an equivariant target, the required condition is:

`y(T_g x) = rho_Y(g) y(x)`.

Numerical tolerance must be frozen from the solver and normalization contract
before any model result is inspected. If the condition is not satisfied, the
proposed group is rejected rather than repaired by post-processing.

### 3.2 Quantum model

Let `rho_in(g)` be the representation of `G` on the encoded quantum state,
`rho_0` the initial density matrix, `E_phi(x)` the classical-to-quantum
encoding, `U_theta` the trainable circuit, and `O` the measurement observable.
The scalar prediction is:

`f_theta(x) = Tr[ O U_theta E_phi(x) rho_0 E_phi(x)^dagger U_theta^dagger ]`.

The required prepared-state covariance is:

`E_phi(T_g x) rho_0 E_phi(T_g x)^dagger = rho_in(g) E_phi(x) rho_0 E_phi(x)^dagger rho_in(g)^dagger`

up to the declared numerical tolerance. A scalar invariant ansatz must satisfy
`[U_theta, rho_in(g)] = 0` and `[O, rho_in(g)] = 0` for all generators used by
the audit. If the output is equivariant rather than scalar, the output
representation and measurement map must be specified explicitly.

For development rows `(x_i,y_i)`, the empirical loss is:

`L(theta) = (1/N) sum_i ell(f_theta(x_i), y_i)`.

The loss, target standardization, folds, seeds, and safety interpretation must
remain those of the original pipeline unless a separate deviation is accepted.

### 3.3 Initialization hypothesis

The proposed Lie-theoretic initialization is only an empirical hypothesis:

`Var(theta_a) = 2 / dim(g_local)`.

Before using it, the protocol must define the generator normalization, for
example `Tr(T_a T_b) = kappa delta_ab`, the local subalgebra, parameter units,
and circuit ordering. The initialization is rejected if its observed gradient
variance or failed-run rate is not better than the standard-initialization
ablation under the same budget. No theorem is claimed from this formula.

### 3.4 NPID hypothesis

If NPID is included, its exact update must be frozen before fitting. A generic
candidate form is:

`e_t = L_t - L_{t-1}`

`u_t = K_P e_t + K_I sum_{j<=t} e_j + K_D(e_t - e_{t-1})`

`theta_(t+1) = theta_t - alpha * u_t * grad_theta L_t`.

The elementwise operation, anti-windup rule, derivative filter, clipping,
shot-noise handling, and query budget are part of the implementation freeze.
NPID is a controller hypothesis; it does not imply that barren plateaus are
removed or that a better minimum exists.

### 3.5 Resource model

For each method report qubits, circuit depth, one- and two-qubit gate counts,
shots, number of measurements, state-preparation cost, data-loading cost,
transpilation cost if any, optimizer evaluations, peak memory, wall time, CPU
time, and classical simulation cost. The initial local envelope remains q=4,
q=6, and q=8 on the reference CPU-only laptop. No hardware claim is allowed.

Sample efficiency must be defined as `N_method(epsilon, delta)`: the number of
independent training samples needed to reach error tolerance `epsilon` with
confidence `1-delta`, estimated from preregistered learning curves. The phrase
`O(sqrt(N))` is not accepted as a result without a theorem or a controlled
scaling experiment.

## 4. Stage-gated workflow

### Stage 1 - Symmetry audit

Derive candidate transformations from the actual cislunar equations. Transform
all vector states, ephemerides, frames, thrust constraints, and labels rather
than rotating only the spacecraft state. Numerically verify invariant or
equivariant residuals on a predeclared development sample. Record the group,
representations, tolerances, failed transformations, and counterexamples.

**Rejection:** no exact task symmetry, unresolved label transformation, or
symmetry that exists only after changing the original task.

### Stage 2 - Development dataset construction

Construct a small development dataset by rerunning the existing data-generating
equations under the verified transformation. Do not fabricate labels by
rotating features alone. Preserve group identity, original splits, and source
provenance. No calibration or final-test rows may be read.

### Stage 3 - Baseline reproduction

Reproduce C06 and implement a classical equivariant baseline using the same
transformed rows, splits, feature budget, parameter budget, tuning budget, and
resource accounting. Suitable candidates include an E(n)-equivariant graph
model or a representation-theoretic classical model, but the exact choice must
be frozen before any PQC result is viewed.

**Rejection:** baseline reproduction failure, an unverified classical control,
or an unfair data/compute budget.

### Stage 4 - Minimal equivariant PQC ablations

Run only after Stages 1-3 pass. Compare:

1. equivariant PQC versus parameter-matched classical equivariant model;
2. equivariant PQC versus the identical PQC without symmetry constraints;
3. Lie-theoretic initialization versus standard initialization;
4. NPID versus Adam and SPSA with identical objective-evaluation budgets.

All variants use identical rows, splits, seeds, metrics, stopping rules, and
outer-test access. The quantum model is not allowed a larger hyperparameter
search or a hidden classical feature extractor.

### Stage 5 - Resource and statistical analysis

Report training, validation, and test metrics where authorized; seed-wise mean,
median, standard deviation, confidence intervals, effect sizes, and failed-run
counts; gradient norm and variance; learning curves versus sample count; and
all quantum, classical, data-loading, and simulation costs.

Any candidate claim must beat C06 and the matched classical equivariant control
under the existing strict 5% development criterion, with the preregistered
paired uncertainty interval and no safety regression. A sample-efficiency or
runtime claim needs its own frozen scaling analysis and cannot be inferred from
one system size.

### Stage 6 - Decision

The direction is rejected if the symmetry is not exact, the classical
equivariant model matches it efficiently, the candidate gain disappears under
matched compute, the gain depends on post-outcome tuning, or the resource
overhead dominates the claimed benefit. A negative result is a valid finding
and must record a future improvement outside the active pipeline.

## 5. Required paper-ready figures

No placeholder figure or invented result is created by this discussion. If a
future execution is separately authorized, it must record at least:

- symmetry residuals and counterexample counts;
- baseline reproduction and paired model comparison;
- learning curves versus sample count with `N_method(epsilon, delta)` estimates;
- gradient norm/variance and failed-run distributions;
- ablation comparison for symmetry, initialization, and optimizer;
- circuit, shot, data-loading, simulation, memory, and wall-time resources;
- a claim-boundary figure separating development evidence from Gate 6 and
  hardware claims.

Figure IDs and registry entries must be reserved only after the implementation
freeze. Every generated figure must point to immutable raw evidence and the
source commit.

## 6. Authorization status

D051-C is discussion-only. It authorizes no code execution, new development
fit, calibration/final-test access, hardware/GPU run, Gate 5 reinterpretation,
quantum-advantage claim, or Gate 6 run. An executable P018 protocol would
require a separate decision after the mathematical contract, symmetry audit
plan, classical baseline, thresholds, resource ceiling, stop rules, and figure
registry are frozen.
